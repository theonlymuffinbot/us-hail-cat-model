## What This Produces

This model builds a full **probabilistic hail hazard layer** for the contiguous United States (CONUS):

- **Return period maps** — Hail size (inches) at 10, 25, 50, 100, 200, 250, and 500-year return periods for every 0.25° grid cell (~28 km) across CONUS.
- **Annual occurrence probability rasters** — Probability that hail exceeds 0.25", 0.50", 1.00", 1.50", 2.00", 3.00", 4.00", or 5.00" at each cell in a given year.
- **Event catalog** — Discrete historical storm events (2004–2025) identified using a synoptic-system grouping rule: consecutive hail days are merged if footprints overlap within 83 km and duration ≤ 5 days. Each event records dates, footprint, peak hail, and centroid lat/lon.
- **Daily climatology** — 366 daily climatology rasters capturing seasonal variation.
- **50,000-year stochastic catalog** — Generated by **event-resampling**: each simulated event draws a historical event template (weighted by seasonal proximity) and applies a log-normal intensity perturbation (σ=0.15). Real spatial footprint geometry is preserved. Produces occurrence and aggregate Probable Exceedance Tables (PETs).

The model uses **no commercial hazard data** — only publicly available NOAA SPC storm reports and US Census Bureau population estimates.

## Quick Start

### 1. Clone the repository

```bash
git clone https://github.com/theonlymuffinbot/us-hail-cat-model.git
cd us-hail-cat-model
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

Python 3.9+ required. The pipeline checks all dependencies at startup and fails fast with a clear message if any are missing.

### 3. Run the pipeline

```bash
python run_pipeline.py
```

All 15 stages run in order (~6–7 hours total). Progress and timing printed per stage. If a stage fails, the pipeline stops and tells you how to resume:

```bash
python run_pipeline.py --from 7       # resume from stage 7
python run_pipeline.py --only 10      # re-run a single stage
python run_pipeline.py --skip 14,15   # skip stages
python run_pipeline.py --dry-run      # preview without running
python run_pipeline.py --validate     # check outputs only, no computation
```

**Render maps after stages 10–13:**

```bash
python scripts/render_maps.py            # RP + p_occ maps
python scripts/render_spatial_corr.py   # spatial correlation diagnostics
```

> **Note:** Stages 3, 6, 7, 8, and 14 skip existing files and are safe to re-run after interruption.

## Pipeline

| Step | Script | What it does | Approx runtime |
|---|---|---|---|
| 1 | `01_download_population.py` | Download Census PEP county population estimates (2000–2023) | 2 min |
| 2 | `02_build_population_trend.py` | Fit piecewise log-linear population trend (1980–2023) | 1 min |
| 3 | `03_download_spc.py` | Download NOAA SPC daily hail/wind/tornado CSVs (2004–present) | 5 min |
| 4 | `04_build_storm_trends.py` | Normalize storm counts by county population, compute β | 2 min |
| 5 | `05_build_spatial_beta.py` | Compute spatial neighborhood beta per county | 1 min |
| 6 | `06_build_hail_rasters.py` | Build raw 0.05° daily hail rasters from SPC reports | ~2 hrs |
| 7 | `07_build_hail_debias.py` | Apply population debiasing (β = 2.37) to raw rasters | ~30 min |
| 8 | `08_build_hail_agg.py` | Aggregate debiased rasters to 0.25°; full corruption + CRS scan | ~30 min |
| 9 | `09_build_hail_climo.py` | Build 366-file daily climatology by calendar day | ~5 min |
| 10 | `10_hail_catmodel_pipeline.py` | Identify events (synoptic grouping), fit CDFs, spatial correlation | ~10 min |
| 11 | `11_build_smooth_cdf.py` | Spatially-pooled CDF rebuild (150 km smoothing) + p_occ threshold TIFs | ~10 min |
| 12 | `12_build_occurrence_probs.py` | Build annual occurrence probability rasters (8 thresholds) | ~5 min |
| 13 | `13_apply_conus_mask.py` | Apply CONUS mask + spatial smoothing to all output rasters | ~5 min |
| 14 | `14_generate_stochastic_catalog.py` | 50,000-yr event-resampling catalog + occurrence/aggregate PETs | ~2.5 hrs |
| 15 | `15_stochastic_maps.py` | Per-cell stochastic RP + p_occ maps from simulation | ~15 min |
| — | `render_maps.py` | Render historical RP + p_occ maps to PNG | ~5 min |
| — | `render_spatial_corr.py` | Render spatial correlation diagnostic figures | ~2 min |

Each stage validates its outputs on completion and exits with code 1 if any files are corrupt or missing, stopping the pipeline chain.

## Known Limitations

- **SPC report density bias:** SPC reports depend on storm spotters and are denser in populated areas. Population debiasing (β=2.37, Step 7) partially corrects this but rural hail may still be undercounted.

- **22-year template library:** The event-resampling stochastic catalog draws spatial footprints from historical events. Novel event geometries outside the 22-year record are not directly representable beyond intensity perturbation. A longer record or MRMS MESH data would improve this.

- **22-year record for CDF fitting:** GPD tail extrapolation beyond ~50-year return periods carries significant uncertainty. L-moments improve stability but cannot substitute for a longer observational record.

- **West coast underrepresentation:** CA/OR/WA have <0.7% of national hail reports. Cells fall below the minimum observation threshold and appear as nodata in RP maps. This is physically correct — the Cascades/Sierra block Gulf moisture.

- **No topographic correction:** Elevation and terrain features affect hail survivability and spotter access. No terrain adjustment is applied.

- **No time-trend correction:** Climate change signals in hail frequency or intensity are not modeled. Stationarity is assumed.

- **Hazard only:** This is a hazard-layer model. Integration with vulnerability curves and an exposure database is required for financial loss quantification.
